1. The Primitive Abstraction

Hal: "First, look at this procedure here, getWeather.

```typescript
function getWeather(city: string): string {
  return JSON.stringify({ ... });
}
```

In the grand scheme, this is what we call a primitive. To the rest of our system, it is a black box. We do not care how it determines the weather. It could be looking at a thermometer, or it could be—as it is here—simply returning a constant truth.

The abstraction barrier is clear: You give it a city, and it returns a string representation of a state. It is a deterministic mapping: f(city)→weather."

2. Metalinguistic Abstraction (Data as Code)

Gerry: "But here is the beautiful part. The Language Model does not know about this function. We must teach it. We must describe the shape of our tool.

Look at the tools array.

```typescript
const tools = [
  {
    type: 'function',
    function: { name: 'get_weather', ... }
  },
];
```

This is data describing a procedure. In Lisp, we often say that code is data and data is code. Here, we are reifying the concept of the function into a JSON structure so that the model can reason about the function without actually executing it yet. We are giving the model a menu of capabilities."

Hal: "Exactly. We are engaging in wishful thinking. We are telling the model: 'Assume you have a procedure called get_weather. If you were to use it, here is what the arguments would look like.'"

3. The Evaluator (The chat Procedure)

Hal: "Now we come to the chat function. This serves as our evaluator.

It takes the state of the world (the messages history) and the available capabilities (tools), and it sends them to the oracle (Ollama).

Notice the signature:

Chat:(State,Tools)→Decision

The response isn't just text anymore. It is a decision. The model effectively decides: 'Do I have enough information to answer? Or must I defer to a tool?'"

4. The Read-Eval-Print Loop (The main Process)

Gerry: "This is where the magic happens. The main function is implementing a control loop, very similar to the standard eval-apply cycle we teach in 6.001.

Let's trace the process evolution:

The Query: We push 'What is the weather in Paris?' onto the stack.

The Evaluation: We call chat. The model analyzes the semantics. It realizes, 'I am a language model; I do not know the weather. But! I have a tool description that does.'

The Deferred Operation: The model returns a tool_calls object. It says, 'I wish to call get_weather with argument Paris.'

Hal: "Note carefully: The model did not call the function. It emitted the intent to call the function. It is up to our main procedure to perform the application."

```typescript
if (response.message.tool_calls?.length) {
    // ... identifying the tool ...
    const result = getWeather(tool.function.arguments.city); // THE APPLY STEP
    // ...
}
```

Gerry: "This is the Apply step. We take the intent, we execute the deterministic primitive getWeather, and we capture the result."

5. Closing the Loop

Hal: "Finally, we must complete the circle. We cannot simply leave the result hanging. We must feed the result of the tool back into the history."

```typescript

```

Hal: "We invoke the oracle a second time. Now, the state has changed.

State 1: User asks about Paris.

State 2: Assistant asks to run tool.

State 3: Tool returns {temp: 22}.

The model sees this history, integrates the tool's output into its linguistic context, and produces the final natural language response: 'The weather in Paris is 22 degrees Celsius.'"

The Robust Implementation

Here is how we modify the system. We introduce a conditional check in our primitive, and we wrap our application step in a try-catch block to reify the exception into a message.

```typescript
/**
 * Single tool with Error Handling.
 * We treat exceptions as valid information for the planner.
 */

const OLLAMA_HOST = process.env.OLLAMA_HOST || 'http://localhost:11434';
const MODEL = 'functiongemma';

// 1. A Fallible Primitive
// We introduce a 'condition' where the procedure fails.
function getWeather(city: string): string {
  if (city.toLowerCase() === 'atlantis') {
    throw new Error(`City '${city}' not found in database.`);
  }
  return JSON.stringify({ city, temperature: 22, unit: 'celsius', condition: 'sunny' });
}

// ... (Tools definition and Chat interface remain the same) ...

const tools = [
  {
    type: 'function',
    function: {
      name: 'get_weather',
      description: 'Get the current weather for a city.',
      parameters: {
        type: 'object',
        properties: {
          city: { type: 'string', description: 'The name of the city' },
        },
        required: ['city'],
      },
    },
  },
];

interface Message {
  role: string;
  content: string;
  tool_calls?: { function: { name: string; arguments: Record<string, string> } }[];
}

interface ChatResponse {
  message: Message;
}

async function chat(messages: Message[]): Promise<ChatResponse> {
  const response = await fetch(`${OLLAMA_HOST}/api/chat`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ model: MODEL, messages, tools, stream: false }),
  });

  if (!response.ok) {
    throw new Error(`HTTP error: ${response.status} ${await response.text()}`);
  }

  return response.json();
}

async function main() {
  // 2. The Wishful Query
  // We intentionally ask for a case that will fail.
  const messages: Message[] = [{ role: 'user', content: 'What is the weather in Atlantis?' }];
  console.log('Prompt:', messages[0].content);

  const response = await chat(messages);

  if (response.message.tool_calls?.length) {
    const tool = response.message.tool_calls[0];
    console.log(`Calling: ${tool.function.name}(${JSON.stringify(tool.function.arguments)})\n`);

    let result: string;
    
    // 3. The Protected Apply
    // We attempt to apply the function. If it fails, we catch the exception
    // and convert it into a string that the LLM can understand.
    try {
      result = getWeather(tool.function.arguments.city);
      console.log('Function Result (Success):', result);
    } catch (error: any) {
      console.log('Function Result (Error):', error.message);
      // We format the error as a JSON object for the model
      result = JSON.stringify({ error: error.message });
    }

    messages.push(response.message);
    
    // 4. Closing the Loop with Failure
    // We send the ERROR back to the model as if it were a valid tool output.
    messages.push({ role: 'tool', content: result });

    const final = await chat(messages);
    console.log('Response:', final.message.content);
  } else {
    console.log('Response:', response.message.content);
  }
}

main().catch(console.error);
```

```typescript
 * Realistic Scenario: Customer Support Triage
 * The model must:

  {

      parameters: {
        type: 'object',
        properties: {
          orderId: { type: 'string', description: 'The alphanumeric order ID (e.g., ORD-123)' },
          reason: { type: 'string', description: 'A concise technical reason for the refund based on the user complaint' },
        },
        required: ['orderId', 'reason'],
      },
    },
  },
  {
    type: 'function',
    function: {
      name: 'track_order',
      description: 'Check the status of a package. Use this for general inquiries about delivery time.',
      parameters: {
        type: 'object',
        properties: {
          orderId: { type: 'string', description: 'The order ID' },
        },
        required: ['orderId'],
      },
    },
  },
];

// ... (Chat and Interface definitions remain standard) ...

// --- The Helper to Execute Tools Dynamically ---
// This is a simple dispatcher pattern.
async function executeToolCall(toolCall: any) {
  const fnName = toolCall.function.name;
  const args = toolCall.function.arguments;

  if (fnName === 'process_refund') {
    return processRefund(args.orderId, args.reason);
  } else if (fnName === 'track_order') {
    return trackOrder(args.orderId);
  } else {
    throw new Error(`Unknown tool: ${fnName}`);
  }
}

// --- Main Process ---

async function chat(messages: any[]): Promise<any> {
    const response = await fetch(`${OLLAMA_HOST}/api/chat`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ model: MODEL, messages, tools, stream: false }),
    });
    return response.json();
}

async function main() {
  // A MESSY, Realistic Input
  // Notice: The user does not say "Call process_refund". They express a situation.
  const userText = "I am absolutely furious! My package (ORD-8821) arrived completely crushed. The box is wet and the item is broken. I want my money back immediately!";
  
  const messages = [{ role: 'user', content: userText }];
  console.log(`User Complaint: "${userText}"\n`);

  const response = await chat(messages);

  if (response.message.tool_calls?.length) {
    const tool = response.message.tool_calls[0];
    
    // HAL: "Observe! The model inferred 'process_refund' from 'crushed' and 'money back'."
    // GERRY: "And look, it extracted 'ORD-8821' from the sentence."
    console.log(`> LLM Decision: Call ${tool.function.name}`);
    console.log(`> Arguments Extracted:`, tool.function.arguments);

    // Execute
    const result = await executeToolCall(tool);
    
    // Feed back
    messages.push(response.message);
    messages.push({ role: 'tool', content: result });

    // Final Natural Language Response
    const final = await chat(messages);
    console.log(`\nAgent Response: "${final.message.content}"`);
  }
}

main().catch(console.error);
```

The Chained Execution Agent

Here is a system that can handle dependencies between tools.

Tool A (list_orders): Gets the context (the history).

Tool B (refund_order): Acts on the data retrieved by Tool A.

```typescript
/**
 * Multi-Step "Agentic" Loop
 * * Scenario: User says "Refund my last order".
 * The model must:
 * 1. Realize it needs an Order ID.
 * 2. Call `get_recent_orders` to find it.
 * 3. Observe the list, pick the last one.
 * 4. Call `process_refund` with that specific ID.
 * 5. Report back.
 */

const OLLAMA_HOST = process.env.OLLAMA_HOST || 'http://localhost:11434';
const MODEL = 'functiongemma';

// --- The "Database" (State) ---
const DB = {
  orders: [
    { id: 'ORD-101', item: 'T-Shirt', price: 20, status: 'delivered' },
    { id: 'ORD-102', item: 'Ceramic Mug', price: 15, status: 'delivered' },
  ]
};

// --- The Primitives ---

function getRecentOrders(userId: string): string {
  // In reality, userId would come from session context
  console.log(`[DB] Fetching orders for user: ${userId}...`);
  return JSON.stringify(DB.orders);
}

function processRefund(orderId: string, reason: string): string {
  console.log(`[BANK] Processing refund for ${orderId}. Reason: ${reason}`);
  return JSON.stringify({ status: 'success', refund_id: 'REF-9000', message: 'Refund processed.' });
}

// --- The Tool Definitions ---

const tools = [
  {
    type: 'function',
    function: {
      name: 'get_recent_orders',
      description: 'Get a list of recent orders for the current user. Returns JSON with order IDs and items.',
      parameters: {
        type: 'object',
        properties: {
          userId: { type: 'string', description: 'The user identifier (default to "current_user")' },
        },
      },
    },
  },
  {
    type: 'function',
    function: {
      name: 'process_refund',
      description: 'Refund a specific order. Requires the exact Order ID.',
      parameters: {
        type: 'object',
        properties: {
          orderId: { type: 'string', description: 'The alphanumeric order ID' },
          reason: { type: 'string', description: 'Why the refund is happening' },
        },
        required: ['orderId', 'reason'],
      },
    },
  },
];

// --- The Dispatcher ---

async function executeTool(name: string, args: any) {
  if (name === 'get_recent_orders') return getRecentOrders(args.userId || 'current_user');
  if (name === 'process_refund') return processRefund(args.orderId, args.reason);
  throw new Error(`Unknown tool: ${name}`);
}

// --- The Recursive Evaluator (The Brain) ---

async function runAgent(messages: any[]) {
  // 1. Ask the Oracle
  const response = await fetch(`${OLLAMA_HOST}/api/chat`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ model: MODEL, messages, tools, stream: false }),
  });
  
  const data = await response.json();
  const message = data.message;

  // 2. Check Termination Condition
  // If the model did NOT ask for a tool, we are done. Return the text.
  if (!message.tool_calls || message.tool_calls.length === 0) {
    console.log(`\n[AGENT] Final Answer: "${message.content}"`);
    return message.content;
  }

  // 3. The Recursive Step
  // The model wants to run a tool. We must run it, update history, and RECURSE.
  console.log(`\n[AGENT] Reasoning... requires tool: ${message.tool_calls[0].function.name}`);
  
  // Append the model's "intent" to the history
  messages.push(message);

  // Execute all requested tools (sometimes it requests multiple parallel calls!)
  for (const tool of message.tool_calls) {
    const fnName = tool.function.name;
    const fnArgs = tool.function.arguments;
    
    console.log(`[EXEC] Running ${fnName} with args:`, fnArgs);
    const result = await executeTool(fnName, fnArgs);
    console.log(`[DATA] Result: ${result}`);

    // Append the "result" to the history
    messages.push({ role: 'tool', content: result, name: fnName });
  }

  // 4. Loop! 
  // We call runAgent again with the NEW history.
  return runAgent(messages);
}

async function main() {
  const userPrompt = "I want a refund for the mug I bought recently. It arrived broken.";
  console.log(`Prompt: "${userPrompt}"`);

  const messages = [{ role: 'user', content: userPrompt }];
  
  await runAgent(messages);
}

main().catch(console.error);
```

```typescript
/**
 * The Dungeon Master Engine
 * * Complexity Level: High
 * Concept: State constraints vs. Narrative freedom.
 * * The LLM acts as the interface. The Code acts as the Dungeon Master.
 */

const OLLAMA_HOST = process.env.OLLAMA_HOST || 'http://localhost:11434';
const MODEL = 'functiongemma';

// --- 1. The World State (The Ground Truth) ---
// This acts as our "Physics". The LLM cannot hallucinate changes here.
type GameState = {
  hp: number;
  location: string;
  inventory: string[];
  doors_unlocked: string[];
};

let STATE: GameState = {
  hp: 100,
  location: 'dungeon_cell',
  inventory: [],
  doors_unlocked: [],
};

// --- 2. The Physics Engine (Tools) ---

// Move: Checks connectivity logic.
function move(direction: string): string {
  if (STATE.location === 'dungeon_cell' && direction === 'north') {
    if (STATE.doors_unlocked.includes('cell_door')) {
      STATE.location = 'guard_room';
      return JSON.stringify({ status: 'success', new_location: 'guard_room', description: 'You enter a room with sleeping guards.' });
    } else {
      throw new Error("The door is locked tight. You cannot phase through matter.");
    }
  }
  return JSON.stringify({ status: 'failure', message: "You bump into a wall." });
}

// Action: Checks inventory constraints.
function unlockDoor(target: string): string {
  if (target === 'cell_door') {
    if (STATE.inventory.includes('rusty_key')) {
      STATE.doors_unlocked.push('cell_door');
      return JSON.stringify({ status: 'success', message: 'The lock clicks open.' });
    } else {
      throw new Error("You do not possess the key for this door.");
    }
  }
  return JSON.stringify({ status: 'failure', message: "That is not lockable." });
}

// Interaction: Modifies state.
function pickup(item: string): string {
  // Hardcoded world items for this demo
  if (STATE.location === 'dungeon_cell' && item === 'rusty_key') {
    STATE.inventory.push('rusty_key');
    return JSON.stringify({ status: 'success', item: 'rusty_key', message: 'Picked up.' });
  }
  throw new Error("You hallucinate the item, but your hands grasp only air.");
}

// --- 3. The Tool Definitions (The Laws of Physics) ---

const tools = [
  {
    type: 'function',
    function: {
      name: 'move',
      description: 'Move the player in a direction (north, south, east, west).',
      parameters: {
        type: 'object', properties: { direction: { type: 'string' } }, required: ['direction'],
      },
    },
  },
  {
    type: 'function',
    function: {
      name: 'unlock_door',
      description: 'Attempt to unlock a specific door.',
      parameters: {
        type: 'object', properties: { target: { type: 'string' } }, required: ['target'],

    },
  },
  {
    type: 'function',
    function: {
      name: 'pickup',
      description: 'Pick up an object from the current room.',
      parameters: {
        type: 'object', properties: { item: { type: 'string' } }, required: ['item'],
      },
    },
  },
];

// --- 4. The Dispatcher ---

async function executeTool(name: string, args: any) {
  try {
    if (name === 'move') return move(args.direction);
    if (name === 'unlock_door') return unlockDoor(args.target);
    if (name === 'pickup') return pickup(args.item);
    return "Unknown action.";
  } catch (e: any) {
    // Crucial: The error is returned to the LLM so it can narrate the failure!
    return JSON.stringify({ status: 'error', reason: e.message });
  }
}

// --- 5. The Game Loop ---

async function gameLoop(history: any[]) {
  const response = await fetch(`${OLLAMA_HOST}/api/chat`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ model: MODEL, messages: history, tools, stream: false }),
  });
  
  const data = await response.json();
  const aiMessage = data.message;

  // If the AI just speaks, print it and wait for user input.
  if (!aiMessage.tool_calls || aiMessage.tool_calls.length === 0) {
    console.log(`\nDM: "${aiMessage.content}"`);
    return; // End turn, wait for next user prompt (in a real app)
  }

  // If the AI wants to act, we enforce the laws of physics.
  console.log(`\n(Internal Monologue: attempting ${aiMessage.tool_calls[0].function.name})...`);
  
  history.push(aiMessage);

  for (const tool of aiMessage.tool_calls) {
    const result = await executeTool(tool.function.name, tool.function.arguments);
    console.log(`(Physics Engine): ${result}`);
    history.push({ role: 'tool', content: result, name: tool.function.name });
  }

  // Recurse: The AI must now describe what happened (success or failure).
  await gameLoop(history);
}

async function main() {
  // The System Prompt sets the stage.
  const systemPrompt = "You are a Dungeon Master. You describe the world. You CANNOT change the world state yourself. You MUST use tools to move, pickup, or unlock. If a tool returns an error, you must describe that failure dramatically.";
  
  // Initial Context
  const history = [

  console.log("\n--- TURN 2: The Correct Sequence ---");
  history.push({ role: 'user', content: "Fine. I pick up the key and use it on the door." });
  await gameLoop(history);
}
```
